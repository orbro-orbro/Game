## 2. 机制教学顺序（首发关卡设计）

> 原则：一次只教一个机制，每个机制用 3 个房间练熟，再加第二个机制。

```
关卡 1：入门 — 15 间

  房间 1~3：纯移动+跳跃
  ├─ 1：平地跑。玩家看到橙色圆点，按左右键移动。
  ├─ 2：跳过一个坑。学会按下跳跃键。
  └─ 3：连续跳过不同高度的平台。用收集品引导路径。

  房间 4~6：引入冲刺
  ├─ 4：水平冲刺。前方有水晶，提示"冲刺穿过去"。
  ├─ 5：空中冲刺。需要跳跃+冲刺才能到达的高台。
  └─ 6：方向冲刺。用上下冲刺避开头顶/脚下的尖刺。

  房间 7~9：引入攀墙
  ├─ 7：垂直墙。前方无路，必须跳向墙壁、抓住、在耐力耗尽前弹跳。
  ├─ 8：两墙间攀跳。左右墙交替，但不可再抓同面墙。
  └─ 9：攀墙+水晶。墙上冲刺到水晶，再抓对面墙。

  房间 10~12：机制组合
  ├─ 10：跳+冲+攀墙连续使用。
  ├─ 11：引入移动平台+冲刺。
  └─ 12：引入碎落平台+攀墙。

  房间 13~15：综合测试
  ├─ 13：长序列，无新元素但要求精确执行。
  ├─ 14：红点藏在偏离主路径的位置。
  └─ 15：小高潮——所有机制连续使用到达终点。
```

## 3. 难度曲线设计

### 3.1 全局难度维度

并非所有"难"都一样。明确每个维度在关卡中的权重：

| 难度维度 | 阶段 1（关 1~3） | 阶段 2（关 4~6） | 阶段 3（关 7~N） |
|----------|-----------------|-----------------|-----------------|
| 操作精度要求 | 低（平台≥3格宽） | 中（≥2格宽） | 高（1格宽/像素级） |
| 反应时间窗口 | 宽松（∞） | 中（移动平台） | 紧（连冲+移动障碍） |
| 机制组合数 | 1~2 | 2~3 | 3 全部 |
| 序列长度 | 短（一屏内 2~3 动） | 中（一屏内 5~7 动） | 长（需跨屏连段） |
| 检查点惩罚 | 0（一屏） | 0 | 0（始终房间级） |

## 5. 红点放置原则

> 红点 = 纯成就收集品，不锁内容。

| 原则 | 说明 |
|------|------|
| 偏离主路径 | 红点不在通关路线上，需要额外操作 |
| 展示而非隐藏 | 让玩家看到红点，思考"怎么过去" |
| 难度略高于主线 | 拿到红点的操作比通关该房间难一档 |
| 每房间 0~2 个 | 平均每间 0.5~1 个，5~8 个/关 |
| 无死亡惩罚 | 拿到红点后死亡，红点仍计入（Celeste 规则） |

## 7. 设计自检问题

每设计一关后检查：

1. 这关引入了什么新概念？前面的关卡教过吗？
2. 玩家在房间 N 失败后，重试路径是否公平？（手感，非视觉）
3. 有没有"假难度"——因为看不清、操作延迟、摄像机问题导致死亡？
4. 红点是否在玩家视线可及但路径稍远处？
5. 连玩 5 间后的操作疲劳度？需要"呼吸房"吗？
