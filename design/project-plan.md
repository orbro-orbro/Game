# 项目管理与里程碑

> 单人开发 | Web | JavaScript | Canvas 2D | 目标：首发 8 关

---

## 1. 开发阶段总览

```
Phase 0      Phase 1       Phase 2        Phase 3        Phase 4        Phase 5
项目搭建    核心移动       进阶机制       关卡系统       内容量产       打磨收尾
 2~3天      1~2周         2~3周          1~2周         3~4周         2~3周

            [====== MVP ======]
            跳+跑+碰撞+单间测试
```

### 1.1 各阶段目标

| 阶段 | 目标 | 产出 | 通过条件 |
|------|------|------|----------|
| **P0** 项目搭建 | 文件结构就绪，浏览器能跑 | index.html + Canvas + 加载链 | 浏览器打开出现黑 Canvas |
| **P1** 核心移动 | 角色能动、能跳、有碰撞 | Grounded/Airborne 两状态 | 一个测试房里跳上平台不穿墙 |
| **P2** 进阶机制 | 冲刺、攀墙全实现 | 5 状态状态机完整 | 能在测试房里完成三角循环 |
| **P3** 关卡系统 | 房间切换、死亡复活、关卡数据 | 关卡 1 可从头打到尾 | 0 BUG 通关关卡 1 |
| **P4** 内容量产 | 关卡 2~8 + 收集品 | 完整 8 关 | 全部可独立通关 |
| **P5** 打磨收尾 | UI、音效、调参、修 BUG | 可发布的完整游戏 | 盲测通关率 ≥ 70% |

## 3. Phase 1：核心移动（MVP 前半）

### 3.1 任务依赖

```
0.5 空场景
  ├─ 1.1 PlayerController 骨架（状态机壳子）
  ├─ 1.2 GroundModule（地面移动+碰撞）
  ├─ 1.3 JumpModule（跳跃+变高+缓冲器）
  ├─ 1.4 PhysicsIntegrator（Raycast 碰撞）
  ├─ 1.5 CameraAnchor（固定镜头）
```

### 3.2 任务详情

| # | 任务 | 核心产出 | 验证方式 |
|---|------|----------|----------|
| 1.1 | PlayerController 骨架 | Grounded/Airborne 状态枚举 + 切换逻辑 | 进 Play 模式不报错 |

| 1.3 | JumpModule | 跳跃 4H 高、16f 到顶、70%/100% 分段、5f 缓冲、5f Coyote | 长按/短按测高度，离开平台 5f 内可跳 |
| 1.4 | PhysicsIntegrator | Raycast 四向检测 + 位置修正 | 撞墙不穿，落地停稳 |

| 1.6 | Test Room 1 | 地板+平台+尖刺+深渊 | 跑跳不死穿 |

## 6. Phase 4：内容量产

### 6.1 关卡生产管线

```
每关 15 间 × 7 关（关 2~8）= 105 间

每间平均耗时（含测试）：30~60 min
总工时估算：105 × 0.75h ≈ 80h

建议节奏：每周 2 关（30 间），约 4 周完成
```

### 6.2 各关清单

| 关 | 主题 | 新元素 | 特殊要求 |
|----|------|--------|----------|
| 1 | 草原 | 无 | P3 已完成 |
| 2 | 森林 | 碎落平台 | 至少 3 间碎落平台房 |
| 3 | 地下 | 移动平台 | 路径移动平台 |
| 4 | 云端 | 冰面 | 冰面 friction 减半 |
| 5 | 雪山 | 高密度尖刺 | 平台宽度 ≤2 格 |
| 6 | 庙宇 | 水晶连冲 | 连冲序列 ≥2 房 |
| 7 | 城市 | 无 | 最窄窗口、最高密度 |
| 8 | 虚空 | 无 | 最长序列、全机制组合 |

### 6.3 红点放置

每关放 5~8 个红点，全 8 关共约 50 个。在房间搭建时顺手放，不作为独立任务。

## 9. 风险清单

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 手感不对——参数需大调 | 高 | 中 | P1 单间测足够久再进入 P2 |
| Raycast 物理有边界情况 | 中 | 高 | P1 多角度跳墙测碰撞 |
| 关卡设计无趣——机制够但节奏差 | 中 | 高 | P3 找人盲测第一关，早发现早修 |
| 内容量产时间超预期 | 中 | 中 | 预设 8 关，但 6 关也可发布（砍关 7、8） |
| 浏览器兼容性 | 低 | 中 | 仅测试 Chrome/Edge（主流），不承诺全浏览器 |

## 10. 砍需求预案

如果时间不够，优先砍：

| 优先级 | 可砍项 | 理由 |
|--------|--------|------|
| 1 | 关卡 7、8 | 6 关≈1.5h 内容，对独立小游戏足够 |
| 2 | 冰面（关 4） | 需要额外摩擦力逻辑+视觉 |
| 3 | 音效 | 纯视觉也可玩 |
| 4 | 关卡选择 UI | 可以 Sequential 自动进下一关 |

**绝不砍的核心**：跳+冲+攀墙三角循环、房间复活、15 间关卡结构、尖刺碰撞。
